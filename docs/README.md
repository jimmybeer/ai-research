## Reading order
1. **Project Overview** – understand why the repo exists and where it is heading.
2. **Contributing Guide** – learn how to structure new prompts, lessons, or automation assets.
3. **Codex Interfaces Guide** – decide which surface to use before starting a task.
4. **Agent System Guide** – load the right agent and know how to extend the system.
5. **Workflow Patterns** – pick a pattern to adapt or expand for current needs.
6. **Getting Started Walkthrough** – complete hands-on exercises to cement the flow.

Revisit this index whenever new documentation is added so you can keep a mental model of the knowledge base.
